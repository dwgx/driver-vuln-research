OVERALL: PARTIAL PASS

The driver loads and creates a device with ZERO access control (any admin
process can open it). However, the PHYS_MAP IOCTL maps physical memory
into the System process address space rather than the calling process.
Direct usermode physical memory read is NOT possible with the simple
map-read-unmap flow.

Root Cause: The driver's ZwMapViewOfSection uses PsInitialSystemProcess
(or the process that DriverEntry ran in) as the target, not the IRP
requestor's process context.

Workaround needed for toolkit integration:
  1. Use IOCTL_MEM_COPY_OUT (0x80002088) after PHYS_MAP_EX to pull data
     from the kernel pool allocation into the IOCTL output buffer
  2. Or use IOCTL_MSR_READ (0x800020EC) which DOES work (confirmed in
     astra64_exploit.py report) for KASLR bypass, combined with a
     different reading strategy
